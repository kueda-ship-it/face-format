# 更新履歴 (CHANGELOG)

## [2026-02-24]
### 計画・設計
- モバイルアプリ化に向けた要件定義と仕様書 (`仕様書.md`) を作成。PWAとCapacitorの2つのアプローチを比較検討の末、既存Push通知との互換性を保つため「手動マニフェスト配置によるPWA化」に決定。
- 開発サーバーをモバイル端末から確認できるよう、ホスト公開オプション (`--host`) を付与して再起動。

### 追加
- PWA（Progressive Web App）としてのインストールに対応。
  - `public/manifest.json` を追加し、`index.html` に関連メタタグを設定。モバイルブラウザから「ホーム画面に追加」を行うことで、ネイティブアプリのような全画面表示が可能になります。

### 修正
- **未完了タスク表示の不具合を修正**: 所属チームを持たないユーザーが「Not Finished」欄で全チームの未完了タスクを閲覧・完了できてしまうバグを修正。管理者以外でチーム選択がない場合は非表示となります。

## [2026-02-18]
### 追加
- プロジェクト仕様書 (`SPECIFICATION.md`) を作成。
- 更新履歴 (`CHANGELOG.md`) を作成。
- 開発サーバーの起動手順を確立。
- マルチプラットフォーム対応（Windows / Apple Silicon）を正式にサポート。

### 変更
- **ナレッジ共有機能の刷新**: インアプリのナレッジ共有機能を削除し、元の外部リンク（GitHub Pages）への遷移に復元しました。
- **外部リンク（ナレッジ共有）の配置最適化**: 「Log Analysis Dashboard」の直上へ移動し、アクセス性を向上させました。
- **投稿フォームの簡素化**: 投稿フォームからタグ選択機能を削除しました。
- 外部リンク（ナレッジ共有）: `https://kueda-ship-it.github.io/Knowledge/`
- 開発サーバーをポート 5173 で起動するように構成（Vite デフォルト）。
- `package.json` から特定のプラットフォームに依存するパッケージ (`@rollup/rollup-darwin-x64`) を削除し、Vite が実行環境に合わせて適切なバイナリを自動選択するように改善。

### 修正
- **ホワイトリスト登録エラーの修正**: 認証前にプロフィールを作成しようとすることによる外部キー制約違反を回避するため、専用の `whitelist` テーブルを導入。初回ログイン時にプロフィールを自動作成するフローに変更。
- **メンション（@タグ）機能の復元**: 欠落していた `useTags` フックの呼び出しを復元し、グループメンションを再度有効化。
- **ダッシュボードの機能整理**: 外部の対応履歴（7万件）分析機能を削除。アプリ本編の投稿・完了状況に特化したシンプルな集計表示に整理。
- **コードのクリーンアップ**: 未使用のインポートや変数に起因する lint 警告を多数解消。
